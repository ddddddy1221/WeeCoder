# Requirement Quality Gate Design

## Goal

把现有“填需求 -> 生成 PRD”的单步能力升级为“需求澄清 -> AI 需求质检 -> PRD 草稿 -> PRD 审批门禁”的可控流程。

## Product Behavior

- 项目经理仍然可以在信息不完整时生成 PRD 草稿，草稿中必须标注缺失项。
- 新增“AI 需求评审”动作，输出完整度、缺失问题、阻塞项、风险提醒和改进建议。
- “提交 PRD 审批”必须满足最低门禁：六个关键需求问题已保存、PRD 草稿已生成、最近一次需求评审通过。
- 项目经理修改任意需求答案后，已有 PRD 和评审结论失效，必须重新评审和重新生成草稿。
- 后台展示业务 skill 清单，让用户看到当前阶段使用了哪些 AI 员工能力。

## Architecture

- 新增共享业务 skill/质检模块，前后端复用同一套 readiness 判断。
- 后端新增 `/api/projects/:projectId/review-requirements`，优先调用 Codex CLI 做 AI 评审，失败时回退到本地规则评审。
- 工作流模型负责写入评审结果、PRD 审批门禁和历史记录。
- 前端在项目经理需求阶段展示质检报告、业务 skill 列表、PRD 草稿生成和审批提交状态。

## Non-Goals

- 本版本不做真实代码开发。
- 本版本不自动创建数据库、服务器、GitHub PR 或 CI 任务。
- 本版本不要求 AI 评审结论百分百准确；它是流程门禁和提示工具，最终仍由负责人审批。
