# Technical Handoff Design

## Goal

把 `PRD 审批 -> 架构与数据设计` 这一步打通：负责人批准 PRD 后，系统基于 PRD 自动生成后续研发交接包。

## Product Behavior

- 当前项目从 `PRD 审批` 推进到 `架构与数据设计` 时，后台自动尝试生成技术交接包。
- 技术交接包包含四类产物：
  - 架构方案：RTSP 接入、YOLO 推理服务、前端视频和标注框、API 契约、异常处理。
  - 开发任务：按前端、后端、推理服务、测试记录拆分任务。
  - 运维需求：运行环境、网络连通、RTSP 凭据、模型服务、日志、监控、重启方式。
  - 测试计划：有行人、无行人、多人、遮挡、弱光、断流、误检率统计。
- 优先调用 Codex CLI 生成；失败时使用本地规则生成可用草稿。

## Non-Goals

- 本版本不直接开发摄像头检测系统代码。
- 本版本不连接真实 RTSP 摄像头。
- 本版本不自动部署 YOLO 模型。
